# Pricing Logic Assumptions

The audit engine uses simplified pricing assumptions for MVP usability and recommendation clarity.

Examples:

* Solo users on “Team” plans are generally considered overspending
* Enterprise plans are assumed unnecessary for very small teams unless explicitly justified
* API pricing recommendations are intentionally conservative because real-world token usage varies significantly
